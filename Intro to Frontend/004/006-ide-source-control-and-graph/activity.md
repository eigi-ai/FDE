# Activity - Use IDE Source Control And Read The Graph

## Goal

Practice source control in the IDE and Git graph reading.

## Task

Use your IDE source control panel and terminal together.

## Requirements

- open repository in IDE
- edit one file
- inspect changed file in Source Control
- stage the file using IDE
- commit using IDE
- run graph command in terminal
- identify `HEAD`
- identify current branch
- identify `origin/main` if available
- explain what Sync means

## Commands

```bash
git status
git log --oneline --graph --decorate --all
```

## Required Notes

Create:

```text
ide-source-control-notes.md
```

Write:

```text
IDE Source Control showed:
Staged files were:
Commit message was:
HEAD pointed to:
Current branch was:
origin/main meant:
Sync means:
```

## Demo Questions

Be ready to answer:

1. What does the Source Control panel show?
2. What is the difference between staged and unstaged in the IDE?
3. What does Sync usually do?
4. What does `HEAD` mean?
5. How do you identify local and remote branches in a graph?
