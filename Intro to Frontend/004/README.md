## Learning Goal

By the end, you should understand:

- what Git is
- what GitHub is
- why developers use Git and GitHub
- how to set up Git
- how to create and clone repositories
- how to stage, commit, and view history
- how to push, pull, fetch, and sync
- what `origin` means
- what local branches, remote branches, and remote-tracking branches are
- how source control works inside an IDE
- how to read a Git graph
- how to work safely without losing code

## Main Mental Model

Keep this flow in mind:

```text
working directory -> staging area -> local repository -> remote repository
```

And this collaboration flow:

```text
my computer <-> GitHub remote <-> teammate's computer
```

Git tracks history. GitHub helps teams share and review that history.
