# Activity - Complete Git And GitHub Workflow

## Goal

Complete a realistic Git and GitHub workflow from local change to GitHub.

## Time

90 minutes

## Task

Create a small repository, push it to GitHub, create a feature branch, and explain the graph.

## Requirements

- configure Git identity
- create or clone a repository
- create a README
- make at least 2 commits
- push to GitHub
- create a feature branch
- make a branch commit
- push branch to GitHub
- open a pull request if possible
- fetch or pull latest changes
- use IDE Source Control
- view Git graph
- explain `origin`, `origin/main`, local branch, remote branch

## Suggested Repository Content

Create:

```text
README.md
git-notes.md
frontend-notes.md
```

## Required Commands

Use at least these commands:

```bash
git status
git diff
git add .
git commit -m "Add notes"
git log --oneline --graph --decorate --all
git remote -v
git push
git fetch origin
git pull
git switch -c feature/git-summary
git push -u origin feature/git-summary
```

## Final Report

Create:

```text
git-github-final-report.md
```

## Demo Questions

Be ready to answer:

1. What is Git?
2. What is GitHub?
3. What did you commit?
4. What did you push?
5. What is `origin`?
6. What is `origin/main`?
7. What is the difference between push, pull, fetch, and sync?
8. What is a local branch?
9. What is a remote branch?
10. What did the IDE Source Control panel show?
11. What did the graph show?
