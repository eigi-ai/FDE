# Activity - Add Form Validation And User Feedback

## Goal

Practice controlled forms, validation, disabled states, and user-facing feedback.

## Task

Improve one form in your app.

Recommended:

- login form
- profile form
- settings form

## Requirements

- use controlled inputs
- create a form values type
- create a validation function
- validate required fields
- show field-level error messages
- prevent submit when invalid
- disable submit while invalid or submitting
- show loading/submitting text
- show success message
- show failure message if needed
- log validation errors during development

## Suggested Validation Rules

Login form:

- email is required
- password is required

Profile form:

- name is required
- email is required
- role is required

Settings form:

- display name is required
- notification preference is required

## Manual Tests

Check:

- submit empty form
- type only one field
- type all required fields
- submit valid form
- check disabled button behavior
- check loading/submitting text
- check success or error message

## Demo Questions

Be ready to answer:

1. Where is your form state stored?
2. What validation rules did you add?
3. What prevents invalid submit?
4. What does the user see when the form is wrong?
5. What changes while the form is submitting?
