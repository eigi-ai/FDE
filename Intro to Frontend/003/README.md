## Learning Goal

By the end, you should be able to build and explain a more production-shaped React frontend using:

- URL-based pages
- React Router
- protected routes
- shared layout and navigation
- responsive grids and forms
- reusable UI components
- API service files
- Vite environment variables
- user-facing API error handling
- TypeScript props, state, form, and API response types
- form validation
- component tests
- user-flow tests
- production build and preview checks

## Main Flow

Keep this full app flow in mind:

```text
URL -> route -> page component -> layout -> state/form/API logic -> UI feedback -> production build
```

This session is not about adding many random features. It is about organizing a frontend app properly.
