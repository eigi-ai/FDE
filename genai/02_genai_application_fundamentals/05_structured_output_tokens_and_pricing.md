# Structured Output, Tokens, and Pricing

## What Is Structured Output

Structured output means asking the model to return data in a predictable shape
instead of loose text.

Examples:

- JSON
- a schema-based object
- a list of fields
- classification labels

This is important when model output must be consumed by software.

## Why Structured Output Matters

Free text is hard for software to trust.

Structured output helps with:

- form filling
- extraction
- workflow automation
- validations
- tool arguments
- database writes

## Common Ways to Generate Structured Output

### Prompt-only formatting

Example:

```text
Return valid JSON with keys: title, sentiment, summary.
```

This is simple but less reliable.

### JSON schema or response format APIs

Some providers allow passing a schema so the model is guided toward a valid
structure.

### Typed validation in application code

The application validates the response using a schema library such as Pydantic,
Zod, or JSON Schema validators.

This is the safest pattern in production.

## What Are Tokens

Tokens are chunks of text used internally by the model.

They are not always whole words.

Examples:

- a short word may be one token
- a long word may be several tokens
- punctuation also counts

Tokens matter because they affect:

- context window usage
- latency
- pricing

## Input Tokens and Output Tokens

- input tokens: the tokens sent to the model
- output tokens: the tokens generated by the model
- cached tokens: reused prompt content that some providers discount
- reasoning tokens: additional internal compute categories exposed by some APIs

## Simple Cost Formula

If a provider charges:

- $A per 1M input tokens
- $B per 1M output tokens

then the rough request cost is:

$$
\text{cost} = \left(\frac{\text{input tokens}}{1{,}000{,}000}\right)A + \left(\frac{\text{output tokens}}{1{,}000{,}000}\right)B
$$

Example:

- input tokens: 20,000
- output tokens: 2,000
- input price: $5 / 1M
- output price: $15 / 1M

Then:

$$
\text{cost} = \left(\frac{20{,}000}{1{,}000{,}000}\right)5 + \left(\frac{2{,}000}{1{,}000{,}000}\right)15 = 0.10 + 0.03 = 0.13
$$

So the request costs about `$0.13`.

## How to Reduce Token Cost

- shorten system prompts
- keep only relevant message history
- summarize long conversations
- retrieve only the top useful documents
- trim tool output
- request concise answers when possible

## Production Rule

Track token usage per:

- request
- feature
- user
- session
- model

That is how teams control cost instead of guessing.
